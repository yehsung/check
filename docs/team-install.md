# aing-check 설치 · 사용 안내 (팀원용)

`aing-check`는 Mac 상단바(메뉴바)에서 근무 상태와 타이머를 확인하는 작은 앱입니다. 아래 순서대로만 따라 하면 됩니다.

## 1. 설치하기

### 방법 A — brew (권장)

**터미널**(Spotlight에서 "터미널" 검색)을 열고 아래 한 줄을 붙여넣고 엔터:

```sh
brew install yehsung/check/aing-check
```

앱은 Apple 공증이 되어 있어 경고 없이 바로 열립니다. 나중에 새 버전이 나오면 `brew upgrade --cask aing-check` 한 줄로 업데이트됩니다.
(brew 가 없다면 [brew.sh](https://brew.sh) 의 한 줄 설치를 먼저 실행하거나, 아래 방법 B를 쓰세요.)

### 방법 B — zip 직접 받기

1. [GitHub Releases](https://github.com/yehsung/check/releases)에서 최신 `aing-check.zip`을 받거나 전달받은 zip을 더블클릭해 풉니다.
2. `aing-check` 폴더 안의 `aing-check.app`을 **응용 프로그램** 폴더로 옮기고 더블클릭하면 열립니다(공증되어 있어 보안 경고 없음).
3. 혹시 예전에 전달받은 미공증 zip이라 **"손상되었기 때문에 열 수 없습니다"** 가 뜨면: 동봉된 `설치하기.command`를 더블클릭하거나(경고로 안 열리면 우클릭 → 열기), 터미널에서 아래 한 줄을 실행합니다.
   ```
   xattr -dr com.apple.quarantine /Applications/aing-check.app
   ```

설치가 끝나면 화면 오른쪽 위 상단바에 aing-check 아이콘이 나타납니다. 아이콘을 클릭하면 작은 창(팝오버)이 열립니다.

## 2. 처음이라면: 가입

상단바 아이콘을 클릭해 창을 연 뒤:

1. **별명**: 팀원 목록에 보일 이름(예: 홍길동)
2. **이메일**: 본인 이메일 주소 — **실제 사용하는 이메일**을 입력해 주세요 (확인 메일은 없지만, 가짜 이메일로 가입하면 비밀번호를 잊었을 때 복구할 방법이 없습니다)
3. **비밀번호**: 앞으로 로그인에 쓸 비밀번호
4. **팀 코드**: 팀에서 전달받은 참여코드를 입력합니다. 맞는 코드면 아래에 "✓ 팀 이름"이 떠서 어느 팀에 들어가는지 미리 확인됩니다. (공백이나 대소문자는 신경 쓰지 않아도 됩니다)
5. **가입** 버튼을 누릅니다 — 계정 생성과 팀 합류가 한 번에 끝납니다.

팀 코드가 아직 없다면(팀을 새로 꾸리는 경우) 가입 화면 아래 **새 팀 만들기**를 눌러 팀 이름과 주간 목표시간(팀원 각자의 1인당 목표 — "각자 이번 주 이만큼은 하자")을 넣고 가입하세요. 팀이 만들어지면서 참여코드가 자동 발급되고, 그 코드를 팀원들에게 전달하면 됩니다. 코드는 나중에도 팀 카드의 열쇠 아이콘에서 볼 수 있습니다.

## 3. 이미 계정이 있다면: 로그인

이메일과 비밀번호를 입력하고 **로그인** 버튼을 누릅니다. 별명은 처음 가입 때 정한 것이 그대로 쓰입니다.

## 4. 근무 시작 / 종료

- 창을 열면 오른쪽에 **근무 시작** 버튼이 있습니다. 누르면 타이머가 시작됩니다.
- 근무 중에는 버튼이 **근무 종료**로 바뀝니다. 일을 마치면 눌러서 종료합니다.
- 시작/종료는 팀 전체에 동기화되어, 다른 팀원의 창에도 내 상태가 보입니다.

## 5. 상단바 표시 읽는 법

상단바 아이콘과 옆의 글자로 지금 상태를 알 수 있습니다.

- **캐릭터 아이콘 + 시간(예: 01:23)**: 근무 중이며, **오늘 일한 총 시간**입니다. 쉬었다 다시 시작해도 0부터가 아니라 오늘 누적에서 이어집니다. (한 시간이 넘으면 `시:분`, 그 전에는 `분:초`)
- **일시정지 아이콘 + `오프`**: 근무 종료 상태입니다.
- **구름 느낌표 아이콘 + `대기`**: 서버와 동기화가 아직 안 끝난 상태입니다. 잠시 기다리면 됩니다.

창 안에서는 팀 이름과 함께 각 팀원이 근무 중인지, 현재/이번 주 근무시간이 얼마인지도 볼 수 있습니다. 헤더(내 상태 박스) 아래에는 내 이번 주 진행 바("이번 주 X / 목표시간", 퍼센트)가 있고, 퍼센트 옆 **연필 버튼**으로 팀의 주간 목표시간을 팀원 누구나 바꿀 수 있습니다(모두에게 반영). 각 팀원 행 아래에도 그 사람의 목표 진행 바가 붙습니다(목표를 채우면 옆에 ✓ 표시). 팀 카드의 **열쇠 아이콘**은 팀원 누구나 눌러 참여코드를 보고 복사할 수 있습니다(새 동료 초대용). 팀별 현황(막대 아이콘)에는 이번 주 근무시간이 있는 팀만 표시됩니다. 창 하단에는 아이콘 버튼 4개가 나란히 있습니다: **캐릭터 표시**(사람 아이콘 — 근무중 캐릭터를 켜고 끔), **새로고침**(원형 화살표), **로그아웃**(문에서 나가는 화살표), **앱 종료**(전원 아이콘). 각 아이콘에 마우스를 올리면 한글 툴팁이 떠서 어떤 버튼인지 알 수 있습니다.

근무를 시작하면 화면 오른쪽 위에 작은 캐릭터가 떠서 인사하고, 클릭하면 반응합니다(방해되면 위 **사람 아이콘** 버튼으로 끄면 됩니다). 캐릭터 몸을 클릭했을 때만 캐릭터가 반응하고, 주변 여백 클릭은 뒤에 있는 창으로 그대로 통과되므로 작업을 가리거나 방해하지 않습니다. 캐릭터는 드래그로 원하는 위치로 옮길 수 있고(끄는 방향을 바라보며 따라옵니다), 그 위치는 기억됩니다. 한동안 조용하면 눈을 감고 꾸벅 졸기도 합니다. 쉬는 중에 컴퓨터를 5분쯤 쓰고 있으면 아잉이 "일하는 것 같아서 근무 시작했어요!" 하고 알리면서 **자동으로 근무가 시작됩니다**(잘못 시작됐다면 창에서 근무 종료를 누르면 되고, 같은 제안은 1시간에 한 번만 합니다).

이 맥에 Claude Code/Codex 같은 AI CLI 사용 기록이 있으면 창 맨 아래에 **이번 달 AI 토큰 소모량**이 한 줄로 표시됩니다(콤마 전체 숫자, 마우스를 올리면 상세, 매달 1일 0부터 다시 쌓입니다. 기록이 없는 맥에서는 보이지 않습니다). 그 줄의 사람 아이콘을 누르면 **이번 달 토큰 소모량 순위 페이지**가 열립니다 — 팀별 현황처럼 별도 페이지로, aing-check 사용자 전체가 함께 순위에 뜨고(각자 앱이 자기 사용량을 공유) 인원이 많으면 스크롤로 봅니다.

앱은 로그인할 때 자동으로 실행되도록 처음 한 번 등록됩니다. 원치 않으면 **시스템 설정 > 일반 > 로그인 항목**에서 aing-check를 끄면 되고, 한 번 끄면 다시 켜지 않습니다.

## 6. 자주 묻는 문제

- **연결 오류가 나요 / 상태가 안 바뀌어요**
  서버(무료 플랜)가 한동안 미사용으로 잠시 멈춘 것일 수 있습니다. 담당자에게 "Supabase 프로젝트 Restore가 필요할 수 있다"고 알려 주세요. 복원되면 다시 정상 동작합니다.
- **근무종료를 안 누르고 컴퓨터를 껐어요**
  걱정하지 않아도 됩니다. 신호가 10분 넘게 끊기면 서버가 마지막 신호 시각을 기준으로 자동 근무종료 처리합니다(그때까지 쌓인 오늘 근무시간은 그대로 인정됩니다). 다시 켜면 정상 상태로 돌아옵니다.
- **`다시 로그인 필요`라고 떠요**
  로그인 세션이 만료된 것입니다. 이메일과 비밀번호를 다시 입력해 **로그인**하면 됩니다.
- **`Supabase 키 오류` / `Supabase 키 필요`가 떠요**
  앱 설정 값 문제입니다. 배포 담당자에게 문의하세요.
- **앱을 완전히 끄고 싶어요**
  창 하단의 **앱 종료** 아이콘 버튼(전원 아이콘, 마우스를 올리면 "앱 종료" 툴팁)을 누르면 aing-check가 완전히 종료됩니다. 다시 쓰려면 응용 프로그램 폴더에서 `aing-check`를 다시 실행하면 됩니다.
- **다른 계정으로 바꾸고 싶어요**
  창 하단의 **로그아웃** 아이콘 버튼(문에서 나가는 화살표, 마우스를 올리면 "로그아웃" 툴팁)을 누른 뒤, 다른 이메일/비밀번호로 로그인하면 됩니다.
